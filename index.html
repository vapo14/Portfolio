During the months between May and November of 2019, I participated in the international Genetically Engineered Machine competition (iGEM) as part of team Tec-Chihuahua. We developed a project pursuing the creation of a bio-fungicide to combat a fungus called Verticillium Dahliae, which is commonly found in cotton crops. I initially approached the team as a web developer, but quickly found out that I would be doing so much more. By the end of the project I had learned so much about synthetic biology that I have been interested in bio-informatics ever since. I did end up working on our wiki, which contains every single aspect about our project, but I also worked on different areas, specifically the entrepreneurship part of the project. We researched about how we could create a company based on our product. We took a look at all the legal and financial aspects and wrote a business model that can be found on our wiki. We won Gold Medal, an award for Best Supporting Entrepreneurship, and a nomination for Best Integrated Human Practices. I got to work with HTML, CSS and JS in the form of jQuery. I used the Bootstrap 4 framework, Git and GitHub for version control, and implemented harp.js for compiling our different web pages before uploading them to the iGEM servers. It was an amazing experience that sent me on my path to learning full-stack web development. Go check out our wiki if you want to learn more about our project!

This project started as a necessity. I decided that during my summer of 2020 I would open a small business by providing computer repair services. I did not expect, however, the appearance of the Covid-19 virus, which made me hesitant on whether I should continue this project or not. After small businesses became available again in my country, I decided to carry on. I mostly managed appointments on my Facebook page, but I couldn’t help feeling that I was lacking professionalism, the kind that a website brings to the table. I decided that I would create a website to schedule appointments, where potential clients would be able to view my services as products and “order” one if they wanted to. I had been studying up on the react.js framework and decided to build a web application using the knowledge I had obtained to that point. Implementing React.js, Bootstrap 4 and Firebase, I was able to finish my web app. It is a simple form that any user can interact with in order to create an appointment with my computer repair shop. The data is uploaded to a Firestore database, and a confirmation email is sent to the user.

It serves its purpose just fine, but a new problem arose: How was I going to access the appointments in my database? This question sparked up another aspect of this project: an Appointment Management Dashboard. I am currently working on the first iteration of this project. I have written a Software Requirement Specification document for it, which you can find here. The tools I am using for this project are: React.js, Bootstrap 4, Firebase and Electron.js.

This website is in spanish due to my country of origin. I will upload a translation option ASAP.
